# Labels

Labels mirror the Beyond Data company vocabulary (engineering-relevant subset).

## Type labels

Applied via the `Type` label group.

| Label | Applied when |
|-------|-------------|
| `Type / feature` | New capability |
| `Type / bug` | Defect |
| `Type / chore` | Maintenance, cleanup, refactoring |
| `Type / refinement` | Needs further design before implementation |

## Workflow labels

| Label | Scope | Applied when |
|-------|-------|-------------|
| `epic` | Workspace | Parent issue representing an Epic |
| `task` | Workspace | Standalone unit of work |
| `feature-request` | Workspace | Incoming request to be triaged |
| `ready-for-agent` | Team (`FRA`) | Issue ready for an AFK coding agent |

## Additional labels

None. (No `ui` label and no `client/*` or `sales/*` groups — these are
company-specific and intentionally omitted from the personal workspace.)
